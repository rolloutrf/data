История операций — фундаментальный компонент, обеспечивающий сквозную видимость всех финансовых событий клиента внутри платформы. Является частью финансового менеджмента клиента.

## Задачи

Основная задача — принятие решений на основе анализа агрегированной информации о транзакциях (с возможностью дальнейшей декомпозиции различной гранулярности). Включает в себя атрибуты:

1. Сбор и запись данных  
2. Упорядочивание и хранение  
3. Индексация и поиск  
4. Версионность и неизменяемость  
5. Визуализация  

## Ценность

**Клиент:**

1. Клиент получает единую прозрачную систему аудита транзакций с возможностью проверки любой операции.  
2. Инструмент принятия решений на основе данных (контроль финансов).  
3. Доступ к историческим данным.  

**Бизнес:**

1. Точка входа клиента в PFM.  
2. Источник данных для анализа пользовательского поведения (покупательской способности) для создания доп. ценностей для пользователя.  

## Основные составляющие

### Основные составляющие (b2c)

**Виды транзакций:**

- Онлайн  
- Оффлайн  
- Переводы:  
  - входящие  
  - исходящие  
- Банковские платежи  

**Типы транзакций:**

- оплата товаров и услуг  
- переводы со счета на счет или с карты на карту  
- платежи по счетам  
- автоматические списания  
- снятие наличных денег через кассу банка или банкомат  
- пополнение счета  
- оплата кредитов, штрафов и т. д.  

**Базовая функциональность:**

1. Агрегация транзакционной активности и нормализация этого в общий формат читаемости  
2. Выбор источника транзакций  
3. Индексирование и категоризация  
4. Поиск и фильтрация (дата, мерчант, тип операции, статусы, сумма)  
5. Сортировка по дате  
6. Детализация транзакции (возможность посмотреть метаданные)  
7. Экспорт чеков  
8. Визуализация  
9. Экспорт отчетности  

### Основные составляющие (b2b)

**Виды транзакций:**

- Входящие  
- Исходящие  

**Базовая функциональность:**

1. Транзакционная активность по входящим/исходящим поступлениям  
2. Выбор источника  
3. Структура входящих/исходящих поступлений  
4. Выручка  
5. Сальдо  
6. Аналитика по клиентам  
7. ДДС  
8. Акт сверки  

## Дополнительная функциональность

1. Выявление отклонений  
2. Построение baseline расходов на основе истории  
3. Прогнозирование и планирование  
   - по временным рядам (Prophet, LSTM, ARIMA и т. д.)  
   - прогноз на месяц вперед по категориям  
   - предсказание даты выхода за лимит по бюджету  
   - сезонность, тренды, поведение схожих компаний  
   - рекомендации  


## Метрики

- Adoption rate / Feature RR — с точки зрения проникновения/возврата функциональности в пользователя  
- FrqRate — частота использования функционала / median Spent Time — продолжительность сессии  
- Успешность поиска (?)  
- % завершения базовых сценариев  
- addRevenue — если на основе данных пользовательских транзакций продукт кросс/апп-сейлит + монетизация премиум-функционала  
- % обращений в саппорт (даунгрейд)  
- Качество данных: полнота данных + точность данных + согласованность данных  
- Задержка  
- Время безотказной работы  

## Какие решения есть на рынке

### B2C

#### Сбер

Раздел называется «История».  
Дашборд с основным блоком — история операций, отсортированная от последней даты. Есть фильтры: тип операции, период, карта или счет, сумма.  

Есть общий раздел расходов в текущем календарном месяце. Чипсами разделен на 2 раздела: расходы и зачисления. Круговая диаграмма с топ-3 категориями трат + все остальное; видно общую потраченную сумму за месяц. Можно провалиться в категорию и посмотреть операции по мерчанту и суммам.  

Можно вручную добавить расход.  

#### Т-банк

Раздел «Все операции».  
Дашборд из 3 блоков: траты, доходы, история операций за текущий календарный месяц. Фильтры: период, карта или счет, без переводов.  

В блоки траты и доходы можно провалиться. Есть гранулярность периода: неделя, месяц, год. Представление в виде круговой или столбчатой диаграммы. Под графиком чипсами отображаются топ-6 категорий; при нажатии видно детальную аналитику по мерчанту и историю операций.  

#### Яндекс.Пэй

Раздел «История».  
Список транзакций, фильтры: Карта Пэй, Баллы Плюса, Сплит, Сервисы Яндекса.  

В детальной транзакции: подробная информация о платеже, переход в чат с поддержкой, справка, а по операциям в сервисах Яндекса — список покупок.  

#### Bank of Georgia

Раздел «Statement».  
Дашборд из 3 блоков: траты, доходы, история операций. Фильтры: период, сумма, валюта, карта и/или счет, категории трат. Возможна выгрузка.  

При нажатии на траты или доходы операции фильтруются. В деталях транзакции можно разделить сумму с контактом, поделиться выпиской, изменить категорию, включать/не включать в аналитику.  

### B2B

#### Т-Бизнес

Раздел «Операции».  
Фильтры: все / списания / поступления, период, категории. Видна общая сумма списаний и поступлений.  

В деталях операции: детальная информация, создание платежки/акта/накладной, повтор платежа, выставление счета, запрос ответного платежа.  

#### ВТБ

Раздел «Операции».  
Фильтры: все / списания / поступления, период, счет, поиск; дополнительные: сумма, контрагент, назначение платежа, ИНН, номер документа. Видна общая сумма списаний и поступлений.  

#### Альфа-Бизнес

Раздел «Лента операций».  
Фильтры: период, счет, контрагент, сумма, статус, тип операции, поиск.  

Расширенное отображение операций включает:

- Дата  
- Компания  
- Контрагент  
- Тип операции  
- Статус  
- Сумма  
- Валюта  
- Описание  
- ИНН контрагента  
- Счет контрагента  
- Номер  
- Банк контрагента  
- БИК банка контрагента  
- Корр. счет банка-получателя  
- Банк-корреспондент  
- БИК банка-корреспондента  
- Корр. счет банка контрагента в банке-корреспонденте  
- Платежей в реестре  
- Дата платежа  

В деталях: детальная информация, подписать, повторить, скачать, сохранить шаблон.  

#### Сбербизнес

Два раздела — «Моя аналитика» и «История операций».  

В «Истории операций» фильтры: приход/расход, контрагент, период, статус, сумма, счет, номер документа. Можно импортировать и экспортировать данные по шаблону. В деталях операции: детальная информация, подписать, создать шаблон, скачать, изменить.  

В «Моей аналитике» — график динамики поступлений/списаний по выбранному счету, выбор периода (год или месяц), история поступлений или списаний по контрагентам за текущий календарный месяц.  

#### Точка Банк

Три раздела: «История операций», «Траты по карте», «Аналитика».  

В «Истории операций» фильтры: тип операции, период, счет, карта, категория, поиск.  
Для операций по карте: кому оплачено, с какой карты, скачать справку об операции.  
Для операций по счету: подробная информация о платеже, повторить, поделиться, распечатать, создать шаблон, скачать платежное поручение.  

В «Тратах по карте» — динамика расходов по карте за последний год, сумма транзакций и мерчантов.  

В «Аналитике» — структура входящих платежей и динамика по месяцам, выручка, сальдо, генерация различных отчетов.  


## Какие общие выводы есть по текущим решениям

### B2C
- История операций реализована в виде дашборда со списком транзакций  
- Основная сортировка: по дате, от последней к более ранним  
- Везде есть фильтрация по периоду, карте/счету, у некоторых по сумме, типу операций, категориям  
- Есть разделение истории на траты и доходы  
- У некоторых есть визуализация расходов  
- При переходе в транзакцию у всех банков отображается подробная информация  

### B2B
- Все банки предоставляют фильтрацию операций по периоду, типу (списания/поступления), счету, сумме, контрагенту и другим параметрам  
- У всех банков в истории отображается общая сумма списаний и поступлений  
- Предоставляются расширенные фильтры и более полная детализация операций  
- Есть разделение истории на несколько отдельных «продуктов»  
- Доступны действия с операциями: повторить, создать шаблон, подписать, скачать, поделиться, распечатать  
- Некоторые используют визуальные решения для отображения информации
