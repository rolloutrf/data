Дизайн-система должна:

- ускорять разработку, предоставляя атомарные компоненты, на основе которых собираются все интерфейсы;
- способствовать поддержанию консистентности и узнаваемости продукта;
- быть аскетичной в плане количества вариаций компонентов, если нет веских аргументов для добавления нового функционала.

## Требования

- Поддержка тёмной темы.
- Токенизированная.
- Адаптивная.
- Интерфейсы, собранные с помощью ДС, не должны деградировать при изменениях в компонентах.

## Состав

### Типы компонентов

- Типографика.
- Компоненты лейаута.
- Контролы.
- Иконки.

### Типы токенов

- Цвета.
- Отступы.
- Радиусы.
- Тени.

## Технологии

### Веб

- React.
- TypeScript.
- CSS Modules.

## Другие моменты

Используемые названия должны нести семантическую нагрузку. Например, пресет для кнопки должен быть не `purple`, а `transaction`. Это позволит:

- поддерживать сквозную консистентность;
- эволюционировать визуально без смысловых деградаций в пользовательских сценариях.
